Сегодня на сервере MS SQL где расположены базы 1С была выявлена ошибка 
- 1204 
Экземпляру компонента SQLServerDatabase не удается получить ресурс LOCK в данный момент времени.
Для выяснения причины и устранения проблемы требовалось проверить конфигурацию блокировки и памяти для данного сервера, 
либо выполнить проверку давно выполняющихся транзакций.

Возможны следующие причины этой ошибки.
1.SQL Server не удается получить дополнительную память операционной системы из-за того, что другие процессы используют ее, либо из-за того, 
что для сервера установлено значение параметра max server memory (не правильно разграничена память по потреблению ресурсов).
2.Диспетчер блокировок не может использовать более 60 процентов доступной памяти для SQL Server.

Были подозрения, что диспетчер блокировок использовал максимальное количество свободной памяти. 

Действия которые были предприняты
перенастройка блокировок, 
проверка на включение трейс флагов, 
сколько памяти занято под блокировки, 
не держит ли кто транзакцию с кучей блокировок,
настройки эскалации на таблицах.

Когда выяснелось, что на такой объём растущей базы 280Гб стало явно нехватать выделеной памяти (ресурс памяти расчитывается от объёма базы), 
был перенастроен ресурс подребляемой памяти.
Когда завершились все настройки по ресурсу блокировок и потребляемой памяти был перезагружен сервер.
В итоге на данный момент сервер работает без сбоев.
